Practical Exercise 7 - Load balancer

 

Task 1

Now we want to build a load balancer scenario with an nginx based load balancer and three colored websites website1 (blue), website2 (red) and website3 (green). The websites (as well as the underlying database) are taken form practical exercise 6, task 6.

Modify the compose file from practical exercise 6, task 6 such that the three websites can be accessed by ports 9711, 9712 and 9713. Test your three colored websites!

 

Task 2

Write the docker file for the nginx load balancer using the official build of nginx. Because the load balancer is configured by nginx.conf, copy this file into the container.

The load balancer shall listen at port 8080. The simplest configuration for nginx.conf is as follows:

events { worker_connections 1024; }
http { sendfile on;
       upstream websites {
         server website1;
         server website2;
         server website3; }
server { listen 80;
  location / {proxy_pass http://websites;}
       }
     }

Test your load balancer! Which balancing algorithm does it use? Here you can see my result: Load Balancer. Which balancing algorithm do I use?

Hint: Writing docker-compose don't forget to provide php for nginx.

To submit to LEA: All php files packed in a zip.

 

Task 3

Read Using nginx as HTTP load balancer. Please explain the following nginx balancing algorithms:

  • Default round robin load balancing.
  • Weighted load balancing.
  • Least connected load balancing.
  • Session persistence load balancing.

Try to test the different algorithms with our test scenario. Is it possible to test all of these methods by our docker scenario? If possible, describe feasible test cases for these tests.

To submit to LEA: Detailed description of feasible test scenarios.
